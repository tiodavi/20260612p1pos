一、請以時間複雜度之漸近符號觀念回答下列問題。(一)、(二)題為二個程式片段分別以漸近符號（$\Theta$）與 n 表示其時間複雜度。(時間複雜度之漸近符號分 $O$、$ \Omega$、$\Theta$ 三種，定義不同，(一)、(二)題請以 $\Theta$ 回答)。
(一) 程式片段如下：(5分)
sum = 0;
for (i = 0; i < n; i = i + 2)
    for (j = 0; j < i; j++)
        sum = sum + 1;

【擬答】

1. 內外迴圈執行次數分析：

  • 外層迴圈控制變數 i 從 0 開始，每次遞增 2，其值依序為：$0, 2, 4, 6, \dots$，直到小於 $n$ 的最大偶數。
  • 內層迴圈執行次數由當前的 i 值決定，精確執行次數為 i 次。

2. 總執行次數與複雜度計算：

假設 $n$ 為偶數，則外層迴圈共執行 $n/2$ 次。內層主要基本操作 sum = sum + 1 的總執行次數為各次外層迴圈 i 值的等差級數累加：

$$ \sum_{k=0}^{\frac{n}{2}-1} 2k = 2 \cdot \frac{(\frac{n}{2}-1)(\frac{n}{2})}{2} = \frac{n^2}{4} - \frac{n}{2} $$

取其最高階項並忽略常數項係數，其漸近時間複雜度為：$\Theta(n^2)$。

(二) 程式片段如下：(5分)
sum = 0; i = 0;
while (i < n) {
    for (j = i; j <= n; j++)
        sum = sum + 1;
    i = i + 1;
};

【擬答】

1. 迴圈執行結構分析：

  • 外層 while 迴圈：控制變數 i 從 0 開始，每次加 1，共執行 $n$ 次（$i$ 範圍自 $0$ 至 $n-1$）。
  • 內層 for 迴圈：當外層變數為 i 時，內層變數 j 從 i 執行到 $n$（含邊界），執行次數為 $(n - i + 1)$ 次。

2. 總執行次數與複雜度計算：

將每次外層迴圈對應的內層執行次數進行加總：

$$ \sum_{i=0}^{n-1} (n - i + 1) = (n+1) + n + (n-1) + \dots + 2 = \frac{(n+1 + 2) \cdot n}{2} = \frac{n^2 + 3n}{2} $$

取其最高階項並忽略低階項與常數，該程式片段之漸近時間複雜度為：$\Theta(n^2)$。

(三) 若甲、乙二種不同演算法可解決同一問題，甲的時間複雜度為 $O(n \log n)$，乙的時間複雜度為 $O(n^2)$。將此二演算法以程式語言實作，請列舉一類型測試資料，並說明其會使甲演算法程式實際執行時間不一定比乙演算法程式實際執行時間快。(10分)

【擬答】

雖然大 $O$ 符號（Big-O）在理論上代表當資料量 $n$ 趨近於無窮大時的漸近上界，但在實際硬體環境與特定測試資料下，甲演算法不一定比乙快。以下列舉兩類常見的測試資料情境說明：

1. 測試資料類型一：極小規模的資料量（Small Input Size，$n$ 極小）

  • 說明： 演算法的實際執行時間為 $T(n) = c \cdot f(n) + d$，其中 $c$ 為常數倍率（包含指令複雜度、記憶體存取成本），$d$ 為固定初始化成本。甲演算法常數項或低階項可能較大（例如：過多元數據維護或複雜遞迴）。
  • 實例： 設 $T_{\text{甲}}(n) = 100 \cdot n \log_2 n$，而 $T_{\text{乙}}(n) = 1 \cdot n^2$。當測試資料 $n = 8$ 時：
    • $T_{\text{甲}}(8) = 100 \cdot 8 \cdot 3 = 2400$
    • $T_{\text{乙}}(8) = 1 \cdot 8^2 = 64$
    此時乙演算法的實際執行速度遠快於甲演算法。

2. 測試資料類型二：已接近排序完成（Nearly Sorted）的特定排序資料

  • 說明： 若甲演算法採用未特別優化的快速排序（Quick Sort，平均 $O(n \log n)$），而乙演算法採用插入排序（Insertion Sort，最差 $O(n^2)$）。
  • 實例： 當輸入資料為「已完全排序」或「極接近完成排序」的測試數列時，快速排序可能因基準值（Pivot）選擇不當而退化至最差狀況 $O(n^2)$；反之，插入排序面對此類資料能發揮最佳狀況 $O(n)$ 的效能。在此特定測試資料類型下，乙演算法的實際執行時間將會大幅超越甲演算法。
二、何謂 Complete Binary Tree？假設單一節點的樹其樹高為1，證明若 Complete Binary Tree 含 n 個節點且樹高為 h，則 $2^{h-1} \le n \le 2^h - 1$。(20分)

【擬答】

本題將依申論題標準架構，分為核心定義與數學證明兩部分進行論述：

一、 Complete Binary Tree（完全二元樹）之核心定義

一棵樹高為 $h$ 的完全二元樹（Complete Binary Tree）必須滿足下列兩點結構約束：

  1. 從第 $1$ 層到第 $h-1$ 層的每一層，節點數都必須達到該層的最大容量。即對任意層 $i$ （$1 \le i \le h-1$），其節點數皆為滿載的 $2^{i-1}$ 個。
  2. 在最後一層（第 $h$ 層）的節點不一定要補滿，但所有節點必須由左至右緊密排列，中間絕不可有任何空缺（Gaps）。

二、 節點數 $n$ 與樹高 $h$ 關係之數學證明

欲證明 $2^{h-1} \le n \le 2^h - 1$，可藉由推導高度為 $h$ 的完全二元樹所能容納的「最多節點數（上界）」與「最少節點數（下界）」來完成證實：

(一) 上界推導（最多節點數 $n_{\text{max}}$）：

  • 當完全二元樹的第 $h$ 層全部填滿時，該樹結構即等同於「滿二元樹（Full Binary Tree）」，此時擁有該高度下的極大節點數。
  • 依二元樹性質，各層之最大節點數形成一等比數列，總和計算如下： $$ n_{\text{max}} = \sum_{i=1}^{h} 2^{i-1} = 2^0 + 2^1 + 2^2 + \dots + 2^{h-1} = \frac{1 \cdot (2^h - 1)}{2 - 1} = 2^h - 1 $$
  • 故在高度 $h$ 的限制下，實際節點數 $n$ 必然滿足：$n \le 2^h - 1$ —— (式1)

(二) 下界推導（最少節點數 $n_{\text{min}}$）：

  • 依據定義，高度為 $h$ 的完全二元樹，其第 $1$ 到第 $h-1$ 層必須完全填滿。前 $h-1$ 層的滿載節點總數為：$\sum_{i=1}^{h-1} 2^{i-1} = 2^{h-1} - 1$。
  • 為了使樹高確實達到 $h$，最底層（第 $h$ 層）至少必須包含 1 個節點。
  • 將兩者相加，可得高度為 $h$ 的完全二元樹之極小節點數： $$ n_{\text{min}} = (2^{h-1} - 1) + 1 = 2^{h-1} $$
  • 故在高度 $h$ 的限制下，實際節點數 $n$ 必然滿足：$2^{h-1} \le n$ —— (式2)

三、 結論

綜合 (式1) 與 (式2) 的邊界條件限制，凡樹高為 $h$ 的 Complete Binary Tree，其節點數 $n$ 必然嚴格落於區間內：

$$2^{h-1} \le n \le 2^h - 1$$

經由上述極限狀況分析，本題命題得證。

1. 相鄰矩陣 (Adjacency Matrix)：

  • 演算法說明： 矩陣中某一特定節點（第 $i$ 列）的 Degree 數值，即為該列從第 1 行到第 $n$ 行所有正元素（值為1）的加總。若要求出圖中「所有節點」的 Degree，演算法必須完整走訪與掃描此 $n \times n$ 的整格二維陣列。

  • 演算法說明： 計算所有節點的 Degree，可維護一個長度為 $n$ 的計數陣列並初始化為 0。接著依序走訪整組多重串列中所有的邊節點（共 $e$ 個）。每讀取到一個邊節點，便將其內含的兩個端點欄位 $v1$ 與 $v2$ 於計數陣列中對應的位置加 1。走訪結束後，陣列即為所有節點的 Degree。
  • 時間複雜度： $O(n + e)$ （初始化計數器耗時 $O(n)$，線性走訪所有邊節點耗時 $O(e)$）。

(三) 請分別說明 Binary Search Tree 與 Red Black Tree 在插入、刪除與搜尋數字等三操作的時間複雜度。(12分)

【擬答】

兩資料結構於基本操作之漸近時間複雜度（以大 $O$ 符號表示）全面對比與分析如下：

核心資料結構 操作效能情境分類 搜尋 (Search) 插入 (Insertion) 刪除 (Deletion)
Binary Search Tree (BST) 平均狀況 (Average Case) $O(\log n)$ $O(\log n)$ $O(\log n)$
最差狀況 (Worst Case) $O(n)$ $O(n)$ $O(n)$
Red Black Tree (RBT) 平均狀況 (Average Case) $O(\log n)$ $O(\log n)$ $O(\log n)$
最差狀況 (Worst Case) $O(\log n)$ $O(\log n)$ $O(\log n)$

核心原理與結構分析：

  • 二元搜尋樹 (BST)： 操作耗時完全取決於樹的高度。在隨機數值輸入下，樹高能保持在 $O(\log n)$；但若遭遇如本題第(一)小題之**完全遞增或遞減的極端輸入序列**，其結構將徹底退化為與單向鏈結串列無異的歪斜樹，導致搜尋、插入與刪除之最差時間複雜度皆劇烈退化至 $O(n)$。
  • 紅黑樹 (RBT)： 屬於具備自平衡機制的自平衡二元搜尋樹。其內部透過紅黑五大性質嚴格制約，保證樹中「最長路徑不超過最短路徑的兩倍」。是以，即便遭遇連續遞增等最極端的測試資料，其內部依然能藉由 $O(1)$ 次的旋轉與重繪將整體樹高永久鎖定在 $O(\log n)$ 層。故其三大基本操作不論在平均或最差狀況下，其時間複雜度皆能穩定地控制在極其高效的 $O(\log n)$，具備強健的防退化保護。
